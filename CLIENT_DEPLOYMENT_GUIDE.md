## 1. Prerequisites & Accounts

Before beginning the deployment, ensure you or the client has set up the following developer accounts:

*   **Google AI Studio**: For the Gemini Live API keys.
*   **LiveKit Cloud**: For the real-time audio WebRTC server.
*   **Telnyx Portal**: For purchasing phone numbers and FQDN routing.
*   **Supabase Cloud**: For call logs database and recording storage.
*   **Cal.com** or **Google Calendar**: For managing appointments.
*   **Telegram Bot** (optional): For real-time booking alerts.

## 2. Database & Storage Setup (Supabase)

The Voice Agent uses Supabase to store call logs, real-time transcripts, active call states, and the OGG audio recordings of every call.

### 2.1 Database Schema Execution
Go to the **SQL Editor** in your client's Supabase dashboard and run the following queries:

```sql
-- 1. Create Call Logs Table
CREATE TABLE IF NOT EXISTS call_logs (
    id bigint generated by default as identity primary key,
    created_at timestamp with time zone default timezone('utc'::text, now()) not null,
    phone_number text,
    duration_seconds integer,
    transcript text,
    summary text,
    sentiment TEXT,
    estimated_cost_usd NUMERIC(10,5),
    call_date DATE,
    call_hour INTEGER,
    call_day_of_week TEXT,
    was_booked BOOLEAN DEFAULT FALSE,
    interrupt_count INTEGER DEFAULT 0,
    audio_codec TEXT,
    recording_url TEXT,
    caller_name TEXT
);

ALTER TABLE call_logs ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Enable all actions for service role" ON call_logs FOR ALL USING (true) WITH CHECK (true);

-- 2. Create Real-Time Transcripts Table
CREATE TABLE IF NOT EXISTS call_transcripts (
    id           UUID        DEFAULT gen_random_uuid() PRIMARY KEY,
    call_room_id TEXT        NOT NULL,
    phone        TEXT,
    role         TEXT        CHECK (role IN ('user', 'assistant')),
    content      TEXT,
    created_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_call_transcripts_room  ON call_transcripts (call_room_id);
CREATE INDEX IF NOT EXISTS idx_call_transcripts_phone ON call_transcripts (phone);

ALTER TABLE call_transcripts ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anon insert transcripts" ON call_transcripts FOR INSERT TO anon WITH CHECK (true);
CREATE POLICY "Allow anon select transcripts" ON call_transcripts FOR SELECT TO anon USING (true);

-- 3. Create Active Calls Table
CREATE TABLE IF NOT EXISTS active_calls (
    room_id      TEXT        PRIMARY KEY,
    phone        TEXT,
    caller_name  TEXT,
    status       TEXT        DEFAULT 'ringing',
    started_at   TIMESTAMPTZ DEFAULT NOW(),
    last_updated TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE active_calls ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anon all active_calls" ON active_calls FOR ALL TO anon USING (true) WITH CHECK (true);
```

### 2.2 Storage Bucket Configuration
1. Go to **Storage** in the Supabase Dashboard.
2. Create a new **public** bucket named `call-recordings`.
3. Go to **Settings** → **S3 Connection** and copy the **S3 access keys**, **endpoint URL**, and **region** to use in your `.env` configuration.

## 3. Telephony & LiveKit SIP Setup (Telnyx)

This step automates FQDN connections, FQDN bindings, number mapping, and LiveKit trunk configurations.

### 3.1 Provisioning with `setup_trunk.py`
Configure the Telnyx API Key and LiveKit SIP credentials in the client's `.env` file (see Section 4). Then execute the trunk setup script:

```bash
python setup_trunk.py
```

### 3.2 What the script does:
1. Registers a new FQDN Connection on Telnyx pointing to the client's LiveKit SIP Gateway.
2. Resolves and registers the DNS target for `sip.telnyx.com`.
3. Looks up the purchased phone number in the Telnyx account and links it to the newly created connection.
4. Programmatically provisions the **Inbound SIP Trunk** and **SIP Dispatch Rule** inside LiveKit to forward calls into the agent workspace (`call-*` room prefix).
5. Registers the client's outbound trunk.

## 5. AI Prompt & Persona Customization

To tailor the agent's behavior to the specific client's business, edit `config.json` in the root folder. You can configure:

*   `agent_instructions`: Define their name, company name, scope, FAQ, consultation flow, objections, and constraints.
*   `first_line`: The greeting phrase triggered immediately when a call is connected.
*   `llm_model`: Gemini model selection (e.g., `gemini-2.5-flash` or `gemini-2.5-flash-native-audio`).
*   `tts_voice`: Gemini Live native speaker voice (options: `Aoede`, `Kore`, `Puck`, `Charon`, `Fenrir`).

## 6. Production Deployment (Docker / Coolify)

### 6.1 Docker Execution
We recommend running both the FastAPI Dashboard and the background LiveKit Agent as separate processes under Supervisord using the provided `Dockerfile`.

Build the Docker image:
```bash
docker build -t boldflow-voice-agent .
```

Run the container:
```bash
docker run -d \
  --name boldflow-agent \
  --env-file .env \
  -p 8000:8000 \
  boldflow-voice-agent
```

### 6.2 Supervisor Configuration (`supervisord.conf`)
The Docker container automatically uses the supervisor configuration below to keep both the web service and background worker online:

```ini
[supervisord]
nodaemon=true

[program:fastapi_ui]
command=python ui_server.py
autostart=true
autorestart=true
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0

[program:livekit_agent]
command=python agent.py dev
autostart=true
autorestart=true
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0
```

## 7. Client Delivery & Handover Checklist

Before handing the dashboard URL to the client, perform the following QA checklist:

- [ ] **Branding Check**: Open the dashboard (`http://localhost:8000`) and verify that all logos and references display "BoldFlow Labs".
- [ ] **Telephony Routing**: Place an inbound call to the client's DID number. The agent should answer within 2 seconds.
- [ ] **Multilingual Preset Test**: Click on "Language Presets" in the dashboard. Activate "Hinglish" or "Multilingual" and verify that the agent matches the caller's language input.
- [ ] **Booking Flow**: Tell the agent to book a slot. Provide a name and email. Hang up and verify that:
  - The calendar slot is blocked on Cal.com/Google Calendar.
  - A Telegram alert/webhook is triggered.
  - The booking status is saved in Supabase.
- [ ] **Call Recording**: Confirm that the OGG audio recording appears in the Supabase S3 bucket and is playable via the dashboard call log table.
